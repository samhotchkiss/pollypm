## Summary

`pollypm.service_api` is the stable facade that TUIs, CLIs, and integration tests consume. The live implementation is `PollyPMService` at `src/pollypm/service_api/v1.py`, re-exported by `pollypm.service_api.__init__`. External callers must prefer the facade over importing `pollypm.supervisor.Supervisor` directly — outside of `pollypm.core` and tests, direct Supervisor imports are **deprecated** and the import-boundary test (`tests/test_import_boundary.py`) enforces an allow-list.

`PollyPMService` binds once to a config path and then answers every call against a freshly-constructed Supervisor instance. Methods are grouped into: status / snapshots, account management, session and lease control, per-task worker lifecycle, alerts and heartbeat, itsalive deploys, token ledger, project registration, task-backend operations, and preference patches.

Touch this module when adding a stable API surface that a TUI or a cockpit panel needs. Do not add helpers that are only used once internally — they belong in the module that owns the behavior. Also do not widen the Supervisor surface; extract to a collaborator and let the service call that collaborator.

## File Structure

- `src/pollypm/service_api/__init__.py` — re-exports `PollyPMService` and supporting types.
- `src/pollypm/service_api/v1.py` — the full implementation. Single-file facade by design.
- `tests/test_import_boundary.py` — enforces the Supervisor-import allow-list.
- `tests/test_service_api_v1_*.py` — pin the stable surface.

## Implementation Details

- **Construction cost**: every call builds a fresh Supervisor via `load_supervisor(readonly_state=...)`. This is deliberate — it keeps the facade side-effect-free and makes caller code trivially safe to run concurrently against the same config path. `readonly_state=True` flips the state store into read-only mode so status reads don't race with writers.
- **Allow-list**: `tests/test_import_boundary.py` maintains an explicit set of files that may still `from pollypm.supervisor import Supervisor`. New direct imports outside `pollypm.core` must either go through `PollyPMService` or add themselves to the allow-list with a TODO. v1.1 plans to make the allow-list empty.
- **Shape discipline**: method return types are typed with dataclasses or protocol objects, never raw `dict[str, Any]` except where historical callers require it (e.g., `session_status`). When adding a new method, prefer a dataclass over a dict.
- **What lives here vs. elsewhere**: plumbing that wraps a single collaborator (account manager, projects, token ledger, itsalive) belongs here so the TUI has one stable surface. Pure utilities (`plan_launches_readonly`, `render_json`) are also exported from this module because they pre-date the service facade and callers rely on them.
